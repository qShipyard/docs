narratorlog is configured via environment variables set in your `.env` file (for Docker Compose deployments) or passed directly to the containers. This page lists all variables the API, worker, and web app read at startup.

<Warning>
  `APP_SECRET` and `ENCRYPTION_KEY` **must** be identical between the API and worker containers. If `ENCRYPTION_KEY` differs, the worker will fail to decrypt PATs and AI keys stored by the API.
</Warning>

## Required variables

These variables have no defaults. narratorlog will refuse to start if any of them are missing.

| Variable | Used by | Description |
|---|---|---|
| `DATABASE_URL` | API, Worker | PostgreSQL connection string. Format: `postgresql://user:password@host:5432/dbname?sslmode=disable` |
| `REDIS_URL` | API, Worker | Redis connection string. Format: `redis://host:6379` |
| `APP_URL` | API, Web | Public-facing URL of your narratorlog instance (e.g. `https://narratorlog.example.com`) |
| `APP_SECRET` | API | 32+ character random string for session signing. Generate with `openssl rand -hex 32`. |
| `ENCRYPTION_KEY` | API, Worker | 32+ character key for encrypting stored secrets (PATs, AI keys). **Must match between API and Worker.** Generate with `openssl rand -hex 32`. |

## Optional variables

These variables have safe defaults and only need to be set when you want to override them.

| Variable | Default | Description |
|---|---|---|
| `PORT` | `8080` | Port the API server listens on |
| `LOG_LEVEL` | `info` | Log verbosity. Accepted values: `debug`, `info`, `warn`, `error` |
| `WORKER_CONCURRENCY` | `5` | Number of concurrent background jobs the worker processes |
| `READER_SOCKET` | `/tmp/reader.sock` | Unix socket path for the Rust reader sidecar. Only needed when `ai_depth` is set to `deep`. |

## Plugin credential variables

The following variables are consumed by narratorlog's built-in output plugins. They are read by the plugin subprocesses at delivery time â€” you only need to set the variables for the plugins you actually use.

| Variable | Plugin | Description |
|---|---|---|
| `SLACK_BOT_TOKEN` | `slack` | Slack Bot OAuth token (`xoxb-...`) |
| `NOTION_TOKEN` | `notion` | Notion internal integration token (`secret_...`) |
| `LINEAR_API_KEY` | `linear` | Linear API key |
| `SMTP_HOST` | `email` | SMTP server hostname |
| `SMTP_PORT` | `email` | SMTP server port (default `587`) |
| `SMTP_USER` | `email` | SMTP username |
| `SMTP_PASS` | `email` | SMTP password |
| `SENDGRID_API_KEY` | `email` | Alternative to SMTP: SendGrid API key |

<Tip>
  In Docker Compose deployments, set all of these in `deploy/.env`. The Compose file passes them to all containers that need them.
</Tip>

<Note>
  AI provider keys (Anthropic, OpenAI, Groq, etc.) are stored in the database encrypted, not as environment variables. Configure them in **Settings â†’ AI** in the web UI.
</Note>
